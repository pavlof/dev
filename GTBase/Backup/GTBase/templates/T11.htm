How it works?

This map is pulled into a div (or any other DOM element displayed as block and with an ID) from Google API v3

It's able to display markers and render directions on it, find addresses or set different map options as types, custom icons, controllers displayed...

Features

  • Multiple maps per page
  • Custom controllers
  • Markers rendering
  • Address finder
  • Routes and direction details
  • Custom map settings
  • Methods and callbacks
  • ... ask Pablo for more

Markers Source

Grouptree Maps can get the markers information from HTML-DOM elements or XML documents.

XML example View full XML example

<marker  data-lat="51.539022" data-long="-0.145224" data-infobox="Hello Im the marker1" data-title="Marker1"
          data-heading="140" data-icon="images/marker2.png" data-active-icon="images/marker2-active.png" />

DOM example Notice that you need to write a class in this element to be parsed. You can set this class as an option.

<a href="#" class="map-marker"
         data-lat="51.539022" data-long="-0.145224" data-infobox="Hello Im the marker1"
         data-title="Marker1"  data-heading="140" data-icon="images/marker2.png"
         data-active-icon="images/marker2-active.png" >Marker 1</a>

Parameters

These are the parameters that you can pass to the plugin in the marker source:

  • data-lat="51.539022" - Sets the latitude [required]
  • data-long="-0.145224" - Sets the longitude [required]
  • data-address="40 Belsize park gardens" - This property replaces data-long and data-lat if they don't exist [required if not lat and long]
  • data-infobox="Hello Im the marker1" - Any HTML string to go inside the infobox, remove this attr to don't display infobox
  • data-title="Marker1" - Marker title (displayed on hover)
  • data-heading="140" - Sets the angle from the north for the StreetView
  • data-icon="../images/marker2.png" - icon when unactive
  • data-active-icon="../images/marker2-active.png" - icon when active (if you set data-icon but not data-active-icon won't be any change in the active state fopr the marker icon)
  • data-zoom="18" - zoom to the marker when click

JS Options

gtMaps({
        mapOptions: {
            zoom: 13,
            center: {
                lat: 51.539022,
                long: -0.145224
            },
            mapId: "map-canvas",
            mapType: 0, //0 = road, 1 = satellite, 2 = hybrid, 3 = terrain...
            trafficButton: true,
            panControl: true,
            zoomControl: true,
            scaleControl: true,
            streetViewControl: true,
            overviewMapControl: true,
            mapTypeControl: true
        },
        infobox: {
            boxStyles: {
                background: "#fff",
                width: "160px",
                height: "190px",
                padding: '10px 15px 20px 40px'
            },
            infoBoxOffset: {
                left: 20,
                top: -200
            },
            planRouteHTML: "<a href='#' class='planRoute'>Plan route</a>" //you should keep the class.
        },
        markers: {
            xmlSource: null,
            markerSelector: ".map-marker",
            markerIco: null,
            markerActiveIco: null,
            activeMarkerZoom: 0,
            fitMarkers: true,
            enableStreetView: true,
            streetViewHTML: "<a href='#' class='streeview'>Street view</a>" //you should keep the class.
        },
        addressFinder: {
            searchInputSelector: "#address",
            searchTriggerSelector: "#search",
            addressMarkerIco: "",
            enableDirections: false,
            directionsPanelSelector: "#directions-panel"
        }
    });
Setting Parameter type Default Comment
mapOptions     Sets the options relative to the map, as type, controls...
zoom integer 13 Sets the default zoom when it loads (if there are no markers or marker.fitMarkers is false)
center object { lat: 51.539022, long:-0.145224 } Sets the center of the map (if there are no markers or marker.fitMarkers is false)
mapId string "map-canvas" Set the ID of the DOM element to pull inside the map. Is useful if you want more than one map per page
mapType integer 0 Sets the map type (0 = road, 1 = satellite, 2 = hybrid, 3= terrain)
trafficButton boolean true Displays the button to toggle the traffic view
panControl boolean true Displays the arrows to control the map position
zoomControl boolean true Displays the control for the zoom (doesn't enable or disable the zoom)
scaleControl boolean true Displays the scale
streetViewControl boolean true Displays the control for the yellow guy to drag and show the streetview
overviewMapControl boolean true Displays the small map on the right bottom corner
infobox     Sets the options relative marker's information box
boxStyles object   You can pass any CSS javascript object to set the infobox style
infoBoxOffset object {left: 20, top:100 } Sets the position of the infobox in relation with the marker
planRouteHTML string "<a href='#' class='planRoute'>Plan route</a>" You can pass any HTML string for the plan route trigger (you have to keep the class in the clickable element)
markers     Sets the options relative to the markers and its icons
xmlSource string null Gets as a string the path and filename for the XML source. If you set this it won't get the markers from the DOM
markerSelector ".map-marker" string Sets the jQuery selector to pull the information and parameters for the markers from the HTML-DOM
markerIco string null Sets the default icon for the markers (overwrited by the marker source if is set)
markerActiveIco string null Sets the icon for the markers when are active (overwrited by the marker source if is set)
activeMarkerZoom integer 0 Sets the zoom into the marker when the user activate it
fitMarkers boolean true Enables to display all markers in the map area on load. This property overwrites the mapZoom and mapCenter
enableStreetView bool true Enables to display the trigger to show the streetview of a marker
streetViewHTML string "<a href='#' class='streeview'>Street view</a>" You can pass any HTML string for the streetview trigger in the infobox (you have to keep the class in the clickable element)
addressFinder     Sets the options relative to the address finder, routes and directions
searchInputSelector string "#address"
searchTriggerSelector string "#search"
addressMarkerIco string "" Sets the default icon for the route start and end
enableDirections boolean false Enables the directions from an address to a marker displaying the trigger in the infobox
directionsPanelSelector string "#directions-panel" Sets the jQuery selector to pull the details of the route
