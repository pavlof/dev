How it works?

This plugin validates a all the form items inside an HTML element.

This elements should have a class to detect them as required (you can set this class as an option of the plugin).

Also you can set callback functions when the form is valid or not passed as arguments.

The plugin returns true or false depending if the form has been valid or not, so you can use it in IF conditional statements if you need it.

Features

  • All form elements validation
    • input text
    • textareas
    • radio inputs
    • checkboxes
    • passwords
    • select dropdowns
  • RegEx validations
    • emails
    • telephones
    • postcodes
    • credit cards
    • dates
  • Multiple forms validation
  • Callback functions
  • Custom error displaying layout
  • jQuery UI and GT Form Customizer ready

HTML

To make this plugin work properly you just need to create a well formed form, based on the W3C standar

Text inputs and textareas

<input id="AnyTelephoneField" class="required telephone" type="text" />

You will need to add the class "required" (you can change this class as an option) and if is needed the class of the regEx to apply

Classes that you can apply

  • email
  • email-repeat
  • ccard
  • start-date
  • end-date
  • ccscode
  • postcode
  • telephone
  • number

You can also set the min number of characters in a field with the parameter "data-minchars"

<input class="required" data-minchars="10" type="text" />

If you need to match the value of 2 fields just add the attribute "data-fieldrepeat" with the jQuery selector of the other field

<input id="username" class="required"  type="text" />
<input id="confirmUsername" class="required" type="text" data-fieldrepeat="#username" />

Passwords

<input id="AnyPasswordField" class="required" type="password" />
<input id="AnyPasswordField2" class="required pass-repeat" type="password" />

You wont need any special class for this fields, just required as in any other input

You would use the class "pass-repeat" in a second password field if it's needed

Checkboxes and radio buttons

<input id="Radio1" class="required" type="radio" name="title" value="Mr" />
<input id="Radio2" class="required" type="radio" name="title" value="Ms" />

<input id="Checkbox1" class="required" type="checkbox" name="when" value="Now" />
<input id="Checkbox2" class="required" type="checkbox" name="when" value="Later" />

For this kind of fields it will check (if they have the required class) if one of this fields with the same name has been checked and then there is a value for this group of inputs.

Select dropdowns

<select class="required" >
	<option value="null">Default</option>
	<option value="1">1</option>
	<option value="2">2</option>
</select>

It needs an option with an empty value tag or equals to "null" to check if any option has been selected

$(document).ready(function () {
    $('.submit').click(function () {
        $("#formWrapper").validate();
    });
});

JS Options

validate({
    //classes
    class_error: "error",
    class_mandatory: "required",
    // error messages  
    message_emptyField: "This field is empty",
    message_wrongEmail: "Check your email address",
    message_wrongPostcode: "Check the postcode",
    message_wrongTelephone: "Check the telephone number",
    message_wrongDate: "Insert a date (dd-mm-yyyy)",
    message_wrongCreditcard: "Check the credit card number",
    message_wrongCCSCode: "Check the credit card security code",
    message_differentPass: "The passwords are different",
    message_differentEmails: "The emails are different",
    message_tooShortPass: "The password is too short",
    message_noOptionSelected: "You should select an option",
    message_noBoxClicked: "You should select an option",
    message_differentFields: "Fields are different",
    message_endDateBeforeStart: "The end date should be after the start date",
    message_notenoughcharacters: "The field needs at least ##number## characters",
    message_wrongNumber: "The value should be only numbers",
    //functions
    function_allCorrects: self.submitHandler,
    function_wrongForm: self.errorHandler,
    //parameters
    silent_mode: false,
    min_passLenght: 3,
    checkbox_label_error_class: false,
    error_as_value: true,
    error_html: "##MESSAGE##"
});
Setting Parameter type Default Comment
classes     Sets the options relative to classes of the elements
class_error string "error" Sets the class for the error messages
class_mandatory string "required" Sets the class needed tp check any input
messages     Sets the options relative to error messages displayed
message_emptyField string "This field is empty"
message_wrongEmail string "Check your email address"
message_wrongPostcode string "Check the postcode"
message_wrongTelephone string "Check the telephone number"
message_wrongDate string "Insert a date (dd-mm-yyyy)"
message_wrongCreditcard string "Check the credit card number"
message_differentPass string "Check the credit card security code"
message_differentEmails string "The passwords are different"
message_tooShortPass string "The emails are different"
message_noOptionSelected string "The password is too short"
message_noBoxClicked string "You should select an option"
message_endDateBeforeStart string "The end date should be after the start date"
message_notenoughcharacters string "The field needs at least ##number## characters"
message_wrongNumber string "The value should be only numbers"
callbacks     Sets the functions that will be launched when the form is right or wrong
function_allCorrects object self.submitHandler Function launched when the form is Ok
function_wrongForm object self.errorHandler Function launched when the form has wrong fields
parameters     Sets the functions that will be launched when the form is right or wrong
silent_mode boolean false disable the erros display
min_passLenght int 3 number of chars for a valid password
error_as_value boolean false sets the error as the value of the form or as external HTML
error_html string "##MESSAGE##" sets the error HTML format, only when error_as_value is false. You can write here any HTML element to display the error inside. The error will replace ##MESSAGE##
checkbox_label_error_class bool false adds the error class to the closest label to the checkbox and radio input
